# Licensing handoff

This repository should route serious implementation conversations into written-scope diligence without implying rights or deployment readiness.

## Public handoff language

For scoped diligence:

1. Bring one protected action boundary.
2. Identify the last enforceable hook.
3. Define the first missing-proof HOLD condition.
4. Define the evidence-only bundle needed by reviewers.
5. Move implementation depth, field-of-use, claim mapping, and commercial terms under written scope.

## Public repo should not include

- claim charts
- implementation maps
- target-company analysis
- production APIs
- commercial terms
- field-of-use terms
- patent-license terms

## Why this works

Engineers get a runnable, falsifiable artifact. Counsel gets a clean boundary. Procurement gets a review packet. Licensing depth remains written-scope.
