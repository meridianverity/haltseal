# Public boundary

## Public allowed

- Synthetic evaluation code
- Deterministic canonicalization
- Action digest binding
- Permit token shape
- Signed-head freshness simulation
- HOLD and DENY reason codes
- Evidence-only record examples
- One protected action demo
- Public reviewer guide

## Written-scope only

- Claim charts
- Infringement or reading maps
- Target-company analysis
- Production SDKs or APIs
- Real customer or partner architecture
- Driver, firmware, hypervisor, accelerator, endpoint, or robotics hook implementation depth
- Commercial terms
- Field-of-use terms
- Patent-license terms
- Non-bypassability representations

## Why this boundary matters

The public artifact must be strong enough for engineers to reproduce and critique, while clean enough for legal, security, procurement, and standards reviewers to handle without implying rights, deployment readiness, certification, or conformance.
